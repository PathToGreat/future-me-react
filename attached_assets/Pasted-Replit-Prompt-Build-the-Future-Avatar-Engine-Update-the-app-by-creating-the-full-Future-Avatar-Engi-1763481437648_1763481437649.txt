• activityTrend
• nutritionTrend
• sleepTrend
• stressTrend

• futureBodyCompositionScore
• futureAvatarWidth
• futureAvatarGlow
• futureAvatarPosture
• futureAvatarFace (expression intensity based on stress+sleep trends)
2. Build Future Avatar Rendering
Create a second avatar component called FutureAvatar:
• Same style and shape system as the current avatar
• Uses projected metrics
• Width driven by futureAvatarWidth
• Glow driven by lifestyle score projection
• Expression driven by stress and sleep trend projections
• Posture driven by activity projection
Do not replace the current avatar. This is a second state.
3. Add a Toggle UI
Add a simple UI switch above the avatar section:
Options:
• “Current You”
• “Future You”
Default to “Current You”.
Switching should immediately swap the avatar, the glow effect, the descriptive text, and the small metric labels.
4. Dashboard Integration
Place the toggle + future avatar immediately above the current avatar section.
Order should be:
• ImageUpload (existing)
• Toggle: Current You / Future You
• AvatarContainer (shows one avatar at a time)
• AvatarMetrics (update dynamically when switched)
Keep all styling consistent with your card and gradient system.

Below the avatar, include dynamic explanations:
For future avatar:
• “This is your predicted physical trajectory based on your current patterns.”
• If the future score increases: “You are on track for a stronger, healthier future self.”
• If the score decreases: “Your current patterns may reduce your future potential.”
• If the score is flat: “Your future self will reflect your consistency.”
Use simple conditional logic tied to the 90 day projection.
6. Do Not Break Existing Systems
Preserve all current functionality:
• Authentication
• Firestore listeners
• Trends
• Current avatar logic
• Daily Insight
• Journey Meter
• Zones

Add the future avatar engine as a layer on top of everything.
7. Final Goal
When completed, the user should be able to switch between:
• A real-time avatar based on current habits
• A projected avatar based on lifestyle trends
Both avatars should look visually distinct in a realistic, motivating way.
