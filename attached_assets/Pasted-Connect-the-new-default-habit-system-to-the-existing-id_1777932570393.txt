Connect the new default habit system to the existing identity trait and avatar influence system in a restrained, believable way.

Do not overhaul the avatar. Do not change the visual design of the Habits page. Do not add gamification. This should be a data-flow and mapping upgrade.

Goal:
Default habits should not directly change the avatar one by one. Habits should influence identity traits, and identity traits should influence the avatar. The model should be:

habit completed → trait influence → identity signal → avatar visual influence

Use the existing identity/avatar architecture wherever possible.

---

Core principle:
A single completed habit should not create a dramatic visible avatar change.

Avatar changes should come from repeated patterns over time, especially rhythm/consistency across related habits.

### 2. Create a habit influence aggregator

Create or update a utility that reads completed default habits and converts them into small trait influence values.

Important:

* use recent completion data, not just today
* favor the last 7 days
* optionally include last 14 or 30 days if existing data makes that easy
* normalize the result so habits cannot overpower existing metrics

Suggested restraint:

* habit influence should contribute lightly, roughly 10–20% of the total trait signal at most
* daily metrics should remain the stronger signal for now

Do not let one habit completion create a major trait jump.

---

### 3. Weight by rhythm, not streak hype

Use the existing rhythm count if available, but treat it calmly.

Example logic:

* completed today = small immediate influence
* repeated completion over recent days = slightly stronger influence
* missed days reduce influence gradually
* no punishment language

Use “rhythm” internally, not streak-focused logic.

### 5. Add a visual influence explanation if existing architecture supports it

If the app already shows Visual Influences or avatar explanation cards, include habit-derived influences there.

Example language:

* “Recent sleep and wind-down rhythm are supporting vitality and emotional stability.”
* “Movement and strength habits are contributing to steadier posture and energy.”
* “Reflection and gratitude are supporting emotional steadiness.”

Do not say:

* “Your avatar improved because…”
* “You earned…”
* “You unlocked…”

Keep it observational.

### 6. Update dashboard/insight compatibility only if needed

If the Home dashboard uses Today’s Signals or identity traits, allow completed core habits to contribute to those signals where appropriate.

But do not create a large new dashboard section yet.

Possible subtle language:

* “Recent habit rhythm is contributing to today’s signal.”
* “Core habits are beginning to shape your current direction.”

Only add this if it fits existing components cleanly.

---

### 7. Preserve data safety

Do not duplicate default habit definitions into each user record.

Use:

* defaultHabits config for definitions
* user habitPrefs/defaults for completion/hidden state
* existing identity/avatar functions for derived output

If a user has no default habit data yet, avatar behavior should remain unchanged.

---

### 8. Acceptance criteria

* Default habits now influence identity traits through a restrained mapping
* Repeated habit rhythm has more effect than a single completion
* Habit influence does not overpower existing metrics
* Avatar visual changes remain subtle
* Existing dashboard and avatar logic still works
* Hidden default habits do not contribute unless completed history is intentionally retained by current logic
* Custom habits are not broken
* No gamification language or reward mechanics are added
* No major new UI section is added unless already supported by existing architecture
