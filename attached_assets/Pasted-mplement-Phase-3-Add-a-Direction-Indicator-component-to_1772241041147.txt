Purpose:
Show the user a calm, non-judgmental readout of their overall direction based on trends, not on raw score.
Output format:
Label: “Direction”
Status (one of three):
“Strengthening”
“Stable”
“Declining”
Optional 1-line explanation in grounded tone:
Example:
Strengthening: “Your recent patterns are trending more consistent than your baseline.”
Stable: “Your recent patterns are holding steady.”
Declining: “Your recent patterns have softened compared to your baseline.”
No emojis.
No praise (“great job”), no shame (“bad”), no urgency language.
Data logic:
Use existing logged metrics already displayed on Home:
Sleep, Activity, Stress, Nutrition, Consistency.
Compute a direction score using one of these acceptable approaches (pick the simplest given current code):
Option A (Preferred if you have baseline and 7-day average already):
For each metric, compute delta = (7-day avg - baseline)
Normalize direction impact so that:
Higher is better for Sleep, Activity, Nutrition, Consistency
Lower is better for Stress (invert stress delta)
Sum weighted deltas (weights can be equal in Phase 3 unless you already have weights)
Option B (If baseline is unreliable early on):
Compare last 7 days vs prior 7 days
Same polarity rules (stress inverted)
Use percent change thresholds
Classification thresholds:
Keep thresholds conservative to avoid noisy flipping.
Example:
Strengthening if total_direction_score >= +X
Declining if total_direction_score <= -X
Stable otherwise
Choose X based on your metric scales. The classification should not change day-to-day unless there is a meaningful shift.
